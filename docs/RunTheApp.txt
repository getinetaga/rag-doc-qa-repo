RAG Document QA - Run Guide

This file provides current, cross-platform steps to run the application.
It assumes you are in the repository root.

1) Create and activate virtual environment

	# macOS / Linux
	python3 -m venv .venv
	source .venv/bin/activate

	# Windows (PowerShell)
	python -m venv .venv
	.\.venv\Scripts\Activate.ps1

	# Windows (cmd.exe)
	python -m venv .venv
	.\.venv\Scripts\activate.bat

2) Install dependencies

	pip install --upgrade pip
	pip install -r requirements.txt

3) Configure environment (.env in repo root)

	OPENAI_API_KEY="sk-..."
	HUGGINGFACE_API_KEY="hf_..."
	LLM_PROVIDER=openai
	LLM_MODEL=gpt-4o-mini
	VECTOR_DB_BACKEND=faiss

	# Use these when running PostgreSQL + pgvector
	PGVECTOR_DSN=postgresql://postgres:<password>@localhost:5432/ragdb
	PGVECTOR_TABLE_NAME=rag_embeddings
	PGVECTOR_PRIMARY_SEARCH=pgvector

	Notes:
	- VECTOR_DB_BACKEND supports: faiss, pgvector, hybrid
	- app/config.py auto-loads .env values when variables are not already set

4) Optional: initialize pgvector table

	# macOS / Linux
	psql "$PGVECTOR_DSN" -f scripts/create_pgvector_table.sql

	# Windows PowerShell
	psql "$env:PGVECTOR_DSN" -f scripts/create_pgvector_table.sql

5) Run FastAPI backend

	uvicorn app.main:app --reload

	API base: http://127.0.0.1:8000
	Swagger:  http://127.0.0.1:8000/docs

6) Run Streamlit UI (choose one)

	# Root UI
	streamlit run streamlit_app.py

	# In-process demo
	streamlit run app/streamlit_demo.py

	UI base: http://127.0.0.1:8501

7) Verify quickly

	python -m pytest -q

8) Build capstone presentation (new)

	python scripts/build_capstone_presentation.py

	Generated file:
	rag-doc-qa-repo-clone/docs/RAG_Capstone_Project.pptx

Troubleshooting

- First run may download SentenceTransformers model; this can take time.
- If OpenAI errors reference client/http issues, verify openai/httpx versions from requirements.
- If PostgreSQL data is not visible in pgAdmin, confirm you are connected to the same host/port used in PGVECTOR_DSN.
- If you use PowerShell execution policy restrictions for venv activation, run:
	Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
